# SVG snapshot goldens

Committed `SvgRenderer` output for the 13 curated fixtures in
`../../fixtures/curated/`, rendered under the per-fixture options in
`../../fixtures/curated/render-options.json`. The SVG eval (`../../svg.test.ts`)
byte-compares current output against these files, so any change to `SvgRenderer` —
including a `@dagrejs/dagre` version bump that shifts a coordinate — turns a golden red
and forces a **reviewed** regeneration.

## These are snapshots, not a Python oracle

SVG has **no Python reference**. `prov` ships only a DOT emitter, whose picture is
Graphviz's own layout engine; `SvgRenderer` lays out with **dagre's layered algorithm**,
which does not reproduce Graphviz coordinates (that is the stretch graphviz stage — logged
as **D20** in `DEVIATIONS.md`). So these goldens are `SvgRenderer`'s **own** output,
reviewed once at introduction. Validity is enforced structurally, not by a foreign oracle:

- the corpus sweep in `../../svg.test.ts` renders every one of the 401 documents and
  checks each for no-throw, double-render determinism, a finite `viewBox`, **XML
  well-formedness** (a hand-rolled tag-balance + attribute-quoting walk in
  `../../svg-extract.ts` — no parser dependency), and **absence of external references**
  (no `<image>`/`<script>`/`xlink:href`/`@import`/remote `url(…)`); and
- it cross-checks every glyph's shape + fill + stroke, every bundle rect's fill, every
  tinted relation's edge stroke, and every deduped arrowhead `<marker>` against
  `PROV_THEME`.

## Review note (one-time, at introduction)

All 13 goldens were generated by `SvgRenderer` under `render-options.json` and **rendered
once in a headless Chromium** (each SVG dropped into a sized, checkerboard-backed frame so
transparency and layout were both visible), then eyeballed. Every golden also parses under
a real XML parser (`xml.etree`) as an independent well-formedness cross-check. Highlights
confirmed by eye:

- `primer-triangle.svg` — the three reference glyphs SVG restores that Mermaid could not:
  a **yellow ellipse** entity (`#FFFC87`/`#808080`), a **blue rect** activity
  (`#9FB1FC`/`#0000FF`), and a real **orange house polygon** agent (`#FED37F`). Green
  `wasGeneratedBy`, orange `wasAssociatedWith`/`wasAttributedTo`, each with a matching
  arrowhead marker. `wasAssociatedWith` (plan unset) is a single direct edge — no join
  circle (D18).
- `annotated-entity.svg` — a **folded-corner note** (the fold reads as a turned-down
  top-right corner) holding three `name = value` rows, joined to the entity by a dashed,
  arrowhead-less link.
- `annotated-relation.svg` — the `wasGeneratedBy` relation carries an annotation, so it
  **splits through a small gray join circle**; the first segment (article → circle) is
  **marker-less** and keeps the green label, the second (circle → activity) carries the
  green arrowhead, and the `ex:tool = vim` note hangs off the circle (D18).
- `bundle1.svg` / `bundle2.svg` — the two sub-bundles are **aliceblue rounded rects drawn
  behind their members** (`ex:e1`/`ex:a1` and `ex:ee1`/`ex:aa1`), each labelled top-left;
  the `used` edges inside read `stroke="#8B0000"` (the theme's Graphviz-only `red4`
  projected to its X11 hex by `toCssColor` so it survives an SVG renderer's colour parser,
  D19-style) with the already-legal `red` label colour. The two top-level bundle entities
  sit outside the rects with their `prov:type = prov:Bundle` notes.
- `delegation-chain.svg` — **orange house agents** (`ex:ag1`/`ex:ag2`/`ex:ag3`) and the
  blue `ex:act`; each `actedOnBehalfOf` splits through a join circle with gray `activity`
  legs.
- `member2.svg` — `hadMember` fans from `ex:c` up to two entities, two arrowed edges.
- `nary-derivation.svg` — the full derivation splits through a join circle with gray
  `usage`/`generation`/`activity` legs; the `usage`/`generation` endpoints are gray (no
  PROV-DM domain → `unknown`), `activity` is the blue rect, the entities yellow.
- `communication3.svg` — two **gray** (inferred) activity rects and a black
  `wasInformedBy` arrow — an untinted relation stays black.
- `start1.svg` — the D15 skip case: no drawable content, an honest empty canvas with a
  finite `viewBox="0 0 0 0"`.
- `uselabels.svg` — **two-line labels** (`prov:label` over the identifier as a second
  `<tspan>`) on the entity and activity, plus a `prov:label` note on each.

Interactivity affordances confirmed present in the markup (not re-derivable from a static
screenshot): every element node carries a `<title>` tooltip with its qualified name +
attribute rows and is wrapped in an `<a href="…">` to its URI (SVG2 bare `href`).

**Do not hand-edit a golden.** If output changes, fix the emitter (never the golden),
regenerate with `SvgRenderer` under `render-options.json`, and re-review the diff.
