# Mermaid snapshot goldens

Committed `MermaidRenderer` output for the 13 curated fixtures in
`../../fixtures/curated/`, rendered under the per-fixture options in
`../../fixtures/curated/render-options.json`. The Mermaid eval
(`../../mermaid.test.ts`) byte-compares current output against these files, so any
change to `MermaidRenderer` turns a golden red and forces a **reviewed** regeneration.

## These are snapshots, not a Python oracle

Unlike the DOT goldens in `../python-dot/` (produced by the reference `prov.dot`),
Mermaid has **no Python reference** — `prov` ships only a DOT emitter, and Mermaid's
real renderer is browser-only (Puppeteer is banned from the critical path). So these
goldens are `MermaidRenderer`'s **own** output, reviewed once at introduction. Validity
is therefore enforced structurally, not by a foreign oracle:

- the corpus sweep in `../../mermaid.test.ts` parses every one of the 401 documents'
  output line-by-line (`../../mermaid-extract.ts`), failing loudly on any statement
  form the renderer does not emit; and
- it cross-checks every node shape/class, every `classDef` body, and every tinted
  link's `linkStyle` against `PROV_THEME`.

## Review note (one-time, at introduction)

All 13 goldens were regenerated by `MermaidRenderer` under `render-options.json` and
**eyeballed once** for syntactic plausibility against Mermaid flowchart grammar. The
core forms were additionally sanity-rendered with the `beautiful-mermaid` library
(stadium/rect/hexagon/subroutine/circle shapes, `:::class` attach, `-->|label|` and
arrowless `---|label|` open links, `-.-` dotted links, `subgraph … end`, `classDef`,
`linkStyle`, and `<br/>` line breaks all parse). Highlights confirmed by eye:

- `primer-triangle.mmd` — stadium entity, rect activity, hexagon agent; `linkStyle`
  indices `0/1/2` aligned to the three edges.
- `bundle1.mmd` / `uselabels.mmd` — the two element-annotation dotted links take link
  indices `0` and `1`, so the relation edges start at index `2`; the `linkStyle` lines
  address `2`/`3` (bundle1) and `2` (uselabels) exactly. This is the index-alignment
  invariant under interleaved annotation links. The `used` strokes in `bundle1.mmd` /
  `bundle2.mmd` read `stroke:#8B0000` — the theme's Graphviz-only `red4` projected to its
  X11 hex by `toCssColor` so the color survives a browser's CSS parser (D19); the label
  `color:red` is already valid CSS and stays.
- `delegation-chain.mmd` — two blank circles `b1`/`b2` (the counter advances across
  relations) with gray `activity` legs.
- `nary-derivation.mmd` — the full derivation splits through a circle with gray
  `activity`/`generation`/`usage` legs; `generation`/`usage` endpoints are
  `unknownInferred` (no PROV-DM domain).
- `start1.mmd` — the D15 skip case: a single `flowchart BT` header (no drawable
  content), the honest empty projection.

`click … href "…" _blank` links are emitted for every node URI. The `beautiful-mermaid`
library does not implement the `click` directive (it is interaction-only), but real
Mermaid / GitHub honour it — that is a library limitation, not an output defect.

**Do not hand-edit a golden.** If output changes, fix the emitter (never the golden),
regenerate with `MermaidRenderer` under `render-options.json`, and re-review the diff.
